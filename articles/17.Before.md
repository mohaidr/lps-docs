# before Expressions

`before` is a **pre‑execution hook**: a list of expressions that run **before** an iteration or
request is evaluated. Each expression is evaluated for its **side effects** — typically a
`$find(..., variable=x)` (or any method with a `variable=` output) that stores a variable — and
the returned value is discarded.

The key benefit is **ordering**: `skipIf`, the URL, and headers are all evaluated *before* a
request is sent, so a request cannot set up a variable and use it in its own `skipIf`. A `before`
expression runs **first**, closing that gap — you can prepare data up front and then use it in the
same iteration/request.

`before` is available in two places:

- at **iteration** level (runs once when the iteration starts)
- at **`httpRequest`** level (runs before each request)

## Semantics

- `before` is a YAML **list**; expressions run **in order**, top to bottom.
- Expressions run **before** the matching `skipIf` check.
- A later expression can use a variable an earlier one set.
- Errors are logged and do not abort execution.
- Because it runs on every execution, an **iteration‑level** `before` runs once per iteration,
  while a **request‑level** `before` runs before every request in the iteration.

## Iteration-level before

Set up variables once when the iteration starts, then verify with `skipIf`.

```yaml
name: BeforeDemo
rounds:
  - name: FeatureRound
    numberOfClients: 3
    arrivalDelay: 1000
    runInParallel: false
    iterations:
      - name: getUsers
        httpRequest:
          capture:
            to: users
            as: Json
            makeGlobal: true
          url: https://jsonplaceholder.typicode.com/users
          method: GET

      - name: useMatchedUser
        before:
          - '$find(source=$users.Body, where=item.company.name == "Romaguera-Crona", match=first, variable=matchedUser)'
          - '$find(source=$users.Body, where=item.username == "Bret", select=item.id, match=first, variable=uid)'
        # skipIf skips when TRUE -> this runs only when the match is who we expect.
        skipIf: '${uid} != 1 or "${matchedUser.username}" != "Bret"'
        httpRequest:
          url: 'https://jsonplaceholder.typicode.com/users/${matchedUser.id}/todos'
          method: GET
```

## Request-level before

Use `httpRequest.before` when the setup should run right before that specific request.

```yaml
name: BeforeRequestDemo
rounds:
  - name: FeatureRound
    numberOfClients: 1
    arrivalDelay: 1000
    runInParallel: false
    iterations:
      - name: getUsers
        httpRequest:
          capture:
            to: users
            as: Json
            makeGlobal: true
          url: https://jsonplaceholder.typicode.com/users
          method: GET

      - name: getAlbums
        httpRequest:
          before:
            - '$find(source=$users.Body, where=item.username == "Bret", select=item.id, match=first, variable=userId)'
          url: 'https://jsonplaceholder.typicode.com/users/${userId}/albums'
          method: GET
```

## Notes

- A request cannot use data it **captures itself** — capture happens after the response, while
  `before`/`skipIf` run before the request is sent. Capture the data in an earlier iteration.
- `before` pairs naturally with the [`find`](https://github.com/mohaidr/lps-docs/blob/main/articles/4.Declerative%20Methods.md#find)
  method, but works with any method that stores a `variable=`.
- See runnable examples in [examples/20.Before.md](https://github.com/mohaidr/lps-docs/blob/main/examples/20.Before.md).
