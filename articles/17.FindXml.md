# findxml — Querying XML Responses with XPath

`findxml` filters and projects values out of an **XML** payload using XPath. It is the XML
counterpart of [`find`](4.Declerative%20Methods.md#find) (which works on JSON): select a set of
nodes with `path`, filter them with `where`, pull a value out with `select`, and pick results with
`match`.

Use it when an API returns **XML or SOAP** and you want to grab a value into a variable — usually
inside a [`before` / `after`](Before&AfterExpressions.md) hook.

> **Just want the theory of XML namespaces?** See the from-scratch walkthrough in
> [concepts/findxml-namespaces-explained.md](../concepts/findxml-namespaces-explained.md). This
> article is the practical "how to use it, including the parts that trip people up" guide.

---

## 1. Capture the XML first

`findxml` reads from a captured response body. Capture it as `Xml`, then point `source` at
`$name.Body`:

```yaml
iterations:
  - name: getOrders
    httpRequest:
      capture:
        to: orders          # store the response under "orders"
        as: Xml
        makeGlobal: true
      url: https://example.com/api/orders.xml
      method: GET

  - name: useOrders
    before:
      - '$findxml(source=$orders.Body, path=/orders/order, where=status="shipped", select=id, match=first, variable=firstShippedId)'
    httpRequest:
      headers:
        X-First-Shipped: '${firstShippedId}'
      url: https://example.com/api/next
      method: GET
```

Given this body:

```xml
<orders>
  <order kind="online"><id>1</id><status>shipped</status><total>150</total></order>
  <order kind="store"><id>2</id><status>pending</status><total>50</total></order>
  <order kind="online"><id>3</id><status>shipped</status><total>300</total></order>
</orders>
```

`firstShippedId` becomes **`1`**.

---

## 2. The three query inputs: `path`, `where`, `select`

Think of it like a tiny query over "rows":

| Input | Role | Example |
|-------|------|---------|
| `path` | **Which** elements to grab (the rows) | `/orders/order` |
| `where` | **Filter** applied to each grabbed element | `status="shipped"` |
| `select` | **Which field** to pull from each kept element | `id` |

- **`path`** is an absolute XPath from the document root. Each `/` steps one level deeper.
- **`where`** is an XPath predicate **without the surrounding brackets** — write `where=total>100`,
  not `where=[total>100]`. LPS wraps it for you.
- **`select`** is a **relative** XPath evaluated from each matched node. Omit it to return the
  element's own text.

```yaml
# last shipped order's total -> 300
- '$findxml(source=$orders.Body, path=/orders/order, where=status="shipped", select=total, match=last, variable=lastTotal)'
```

### Attributes use `@`

To filter on or project an **attribute**, prefix it with `@`:

```yaml
# first online order's kind attribute -> "online"
- '$findxml(source=$orders.Body, path=/orders/order, where=@kind="online", select=@kind, match=first, variable=firstKind)'
```

---

## 3. Match modes

`match` decides which of the matched nodes you get back:

| `match` | Result |
|---------|--------|
| `first` (default) | the first match |
| `last` | the last match |
| `index:N` | the N-th match, **0-based** (`index:1` = the second) |
| `count` | how many matched (an integer) |
| `all` | **every** match, as a JSON array |

```yaml
- '$findxml(source=$orders.Body, path=/orders/order, where=status="shipped", match=count, variable=shippedCount)'   # -> 2
- '$findxml(source=$orders.Body, path=/orders/order, where=status="shipped", select=id, match=all, variable=shippedIds)'   # -> ["1","3"]
```

If nothing matches and you supplied `default`, that is stored instead:

```yaml
- '$findxml(source=$orders.Body, path=/orders/order, where=status="cancelled", select=id, match=first, default=NONE, variable=cancelledId)'   # -> NONE
```

---

## 4. Quoting string literals (a common gotcha)

Inside `where` / `select`, write string comparisons with **double quotes**, because the whole
`$findxml(...)` expression is already wrapped in **single** quotes in YAML:

```yaml
- '$findxml(source=$orders.Body, path=/orders/order, where=status="shipped", select=id, match=first, variable=x)'
#                                                            ^^^^^^^^^^^^^^^^  double quotes inside single-quoted YAML
```

Using single quotes inside would prematurely close the YAML string and break parsing.

## 5. XML namespaces — the part that trips people up

If a response declares an `xmlns` (very common, and **always** true for SOAP), a "bare" path like
`/orders/order` will match **nothing**. That is not a bug — XPath 1.0 treats an unprefixed name as
*"no namespace"*, so it won't match elements that *are* in a namespace.

Take this SOAP body:

```xml
<s:Envelope xmlns:s="http://schemas.xmlsoap.org/soap/envelope/">
  <s:Body>
    <o:order xmlns:o="urn:shop:orders">
      <o:id>1</o:id>
      <o:status>shipped</o:status>
    </o:order>
  </s:Body>
</s:Envelope>
```

You have two ways to handle it.

### Option A — declare prefixes with `namespaces=`

Provide a list of `prefix=uri` bindings, then use those prefixes in `path` / `where` / `select`:

```yaml
- '$findxml(source=$soap.Body, namespaces=s=http://schemas.xmlsoap.org/soap/envelope/;o=urn:shop:orders, path=/s:Envelope/s:Body/o:order, where=o:status="shipped", select=o:id, match=first, variable=shippedId)'   # -> 1
```

Syntax rules for `namespaces=`:

- **Bindings are separated by `;`** — never a comma (commas separate the method's parameters, so a
  comma would end the `namespaces` value early).
- **Each binding splits on its first `=`**, so the URI may safely contain `:` and `/`
  (e.g. `s=http://…/envelope/`).
- **The prefixes are yours.** They are matched by the **URI** they point to, not by their letters —
  so they do **not** have to be the same letters the document uses. You could bind
  `soap=http://…/envelope/` and write `/soap:Envelope/...`. The one rule: the prefix you write in
  `path` must be one you declared in `namespaces=`.

### Option B — ignore namespaces with `ignoreNamespaces=true`

Strip every namespace from the document, then use plain, unprefixed XPath:

```yaml
- '$findxml(source=$soap.Body, ignoreNamespaces=true, path=/Envelope/Body/order, where=status="shipped", select=id, match=first, variable=shippedId)'   # -> 1
```

This is the easy button for quick checks when you don't need precision. External entities are
disabled (XXE-safe) on both this and the normal path.

### The default-namespace trap

When a document uses a **default** namespace (an `xmlns` with **no** prefix), there is no prefix to
copy — yet a bare path still matches nothing:

```xml
<order xmlns="urn:shop:orders">
  <id>1</id>
</order>
```

You must **invent** a prefix and bind it (or strip namespaces):

```yaml
- '$findxml(source=$soap.Body, namespaces=o=urn:shop:orders, path=/o:order, select=id, match=first, variable=id)'   # -> 1
#   or:
- '$findxml(source=$soap.Body, ignoreNamespaces=true, path=/order, select=id, match=first, variable=id)'           # -> 1
```

> **Why do I have to provide the mapping even when the document already has prefixes?** Because XPath
> keeps its **own** prefix table and never borrows the document's. The two tables only ever meet
> through the **URI**. For the full mental model, see
> [concepts/findxml-namespaces-explained.md](../concepts/findxml-namespaces-explained.md).

## 6. Typed storage with `as`

The stored variable can be typed. `findxml` defaults to `as=auto` (it infers `int` / `bool` /
`json` / string from the value). Override it when you want a specific type:

```yaml
- '$findxml(source=$orders.Body, path=/orders/order, where=status="shipped", match=count, as=int, variable=shippedCount)'
```

`match=all` stores a JSON array, which downstream methods (or `${var[0]}`) can navigate.

---

## 7. Parameter reference

| Parameter | Required | Description |
|-----------|----------|-------------|
| `source` | yes | The XML to query, usually a captured body: `$name.Body`. |
| `path` | yes | Absolute XPath selecting the node set (the "rows"). |
| `where` | no | XPath predicate applied to each node, **without** brackets. |
| `select` | no | Relative XPath projecting a value from each kept node. Omit for the node's text. |
| `match` | no | `first` (default) \| `last` \| `index:N` \| `count` \| `all`. |
| `default` | no | Stored/returned when nothing matches. |
| `as` | no | Stored type: `auto` (default) \| `json` \| `int` \| `double` \| `decimal` \| `bool` \| `string`. |
| `namespaces` | no | `prefix=uri;prefix=uri` bindings for prefixed XPath. |
| `ignoreNamespaces` | no | `true` strips all namespaces so unprefixed XPath matches. |
| `variable` | no | Name to store the result under. |
| `isGlobal` | no | `false` (default) = session-scoped; `true` = shared across sessions. |

---

## 8. Common pitfalls checklist

- **Everything returns empty against a namespaced/SOAP doc** → you didn't declare `namespaces=` (or
  set `ignoreNamespaces=true`). A bare path can't match namespaced nodes.
- **You reused the document's prefix but it still fails** → the prefix in your `path` must be one
  **you** declared in `namespaces=`, not the one the document happens to use.
- **Default namespace (`xmlns="..."` with no prefix)** → you must invent a prefix and bind it, or use
  `ignoreNamespaces=true`. There is nothing to "copy" from the document.
- **`where` broke the parse** → write string literals with **double** quotes; don't wrap the
  predicate in brackets.
- **Attribute lookups return nothing** → prefix attributes with `@` (`where=@kind="online"`).
- **`namespaces=` only picked up the first binding** → separate multiple bindings with `;`, not `,`.

## 9. End-to-end examples

### Plain (no namespaces)

```yaml
name: FindXmlDemo
rounds:
  - name: FeatureRound
    numberOfClients: 1
    arrivalDelay: 1000
    iterations:
      - name: getOrders
        httpRequest:
          capture: { to: orders, as: Xml, makeGlobal: true }
          url: https://example.com/api/orders.xml
          method: GET
      - name: process
        before:
          - '$findxml(source=$orders.Body, path=/orders/order, where=status="shipped", select=id, match=first, variable=firstShippedId)'
          - '$findxml(source=$orders.Body, path=/orders/order, where=total>100, select=id, match=all, variable=bigOrderIds)'
        httpRequest:
          headers:
            X-First-Shipped: '${firstShippedId}'
          url: https://example.com/api/next
          method: GET
```

### SOAP (with namespaces)

```yaml
name: FindXmlSoapDemo
rounds:
  - name: FeatureRound
    numberOfClients: 1
    arrivalDelay: 1000
    iterations:
      - name: getSoap
        httpRequest:
          capture: { to: soap, as: Xml, makeGlobal: true }
          url: https://example.com/api/orders.soap.xml
          method: GET
      - name: process
        before:
          # declare the SOAP + app prefixes, then address the nodes with them
          - '$findxml(source=$soap.Body, namespaces=s=http://schemas.xmlsoap.org/soap/envelope/;o=urn:shop:orders, path=/s:Envelope/s:Body/o:order, where=o:status="shipped", select=o:id, match=first, variable=shippedId)'
        httpRequest:
          headers:
            X-Shipped-Id: '${shippedId}'
          url: https://example.com/api/next
          method: GET
```
